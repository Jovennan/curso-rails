Estaremos configurando um ambiente de desenvolvimento Ruby on Rails no Ubuntu 16.04 Xenial Xerus. A razão pela qual vamos usar o Ubuntu é porque a maioria do código que você escreve será executado em um servidor Linux. O Ubuntu é uma das distribuições Linux mais fáceis de usar, com muita documentação, por isso é uma excelente para começar. Você vai querer baixar a versão Desktop mais recente aqui: http://releases.ubuntu.com/16.04/ Alguns de vocês podem optar por desenvolver no Ubuntu Server para que seu ambiente de desenvolvimento corresponda ao seu servidor de produção. Obtendo o Gerenciador de Versões do Ruby rbenv

  • $ cd
  • $ git clone https://github.com/rbenv/rbenv.git ~/.rbenv
  • $ echo 'export PATH="$HOME/.rbenv/bin:$PATH"' >> ~/.bashrc
  • $ echo 'eval "$(rbenv init -)"' >> ~/.bashrc
  • $ exec $SHELL

Obtendo o Ruby ruby-build

  • $ git clone https://github.com/rbenv/ruby-build.git ~/.rbenv/plugins/ruby-build
  • $ echo 'export PATH="$HOME/.rbenv/plugins/ruby-build/bin:$PATH"' >> ~/.bashrc
  • $ exec $SHELL
  • $ rbenv install 2.4.0
  • $ rbenv global 2.4.0
  • $ ruby -v
  • $ gem install bundler

IRB

  • $ irb
    O intérprete para Ruby - basicamente, o cérebro principal da linguagem de programação que faz sentido do código que você escreve - lê o código de cima para baixo e da esquerda para a direita; Ou seja, começa na linha 1, o caractere 1, literalmente, e primeiro lê através da linha 1 para o último caractere, então desce até a próxima linha e repete esse processo até atingir a última linha do seu programa.
  • Ruby Objects
    Um objeto é uma coisa. Ele está no coração do Ruby. O Ruby sendo uma linguagem orientada a objetos, significa que Ruby manipula todos os dados na suposição de que os dados são um objeto. Existem muitas linguagens orientadas a objetos, mas poucas colocam o objeto no centro de seu universo como Ruby. Em Ruby, tudo é um objeto. Quero dizer tudo: cada variável, cada operação. Cada objeto tem características diferentes; Isso é o que os torna diferentes. Uma seqüência de caracteres é um objeto que possui características internas que o tornam adequado para manipulação de texto.
  • Ruby Methods

    Considere a variável a seguir:

    string1 = "esta é uma string qualquer"

    Se você escreve string1.capitalize, a saída seria algo assim:

    "Esta é uma string qualquer"
    Um método é simplesmente uma definição de uma ação que pode ser realizada em um objeto. Ruby tem built-in definições de objetos e métodos. Um desses métodos é captilize para a classe string Ruby (vamos mergulhar em seqüências de caracteres mais tarde).

Aula 3 - Testando o Scaffold

Como o Rails gera os artefatos (Models - Cotrollers - Views) automaticamente

Testando o Scaffold

Construindo um novo APP

Abra o terminal e digite o código abaixo, para criar um novo APP

$ rails new junta_medica -d postgresql

Acesse a pasta do APP

$ cd junta_medica

Gere um componente

$ rails generate scaffold patient name:string cpf:string email:string address:string phone:string

Gere o componente Patologia

$ rails generate scaffold pathology description:string cid:integer

Gere o componente Atestado Médico

$ rails generate scaffold medical_diagnostic start_date:date period:integer doctor:string

Edite o arquivo:

config/database.yml

Atualize as informações

username: unipe password: unipe host: localhost port: 5432

Salve

No terminal execute os comandos:

$ rails db:create $ rails db:migrate $ rails server

acesse http://localhost:3000

Estilizando nosso APP

Acesse a pasta do APP:

$ cd junta_medica

abra o editor de código

$ code .
